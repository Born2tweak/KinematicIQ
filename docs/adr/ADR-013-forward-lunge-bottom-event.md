# ADR-013: Forward-Lunge Bottom Event Requires Development Comparison

**Status:** Accepted research constraint.

## Decision

The Phase 3 runtime currently selects bottom from maximum pelvis drop in `web/src/protocols/inlineLunge/segmenter.ts`. Phase 4 requires a preregistered development comparison against maximum lead-knee flexion and event-label evidence before freezing the construct. P4-M00 changes neither thresholds nor event logic.
